KMZ Processor & ImportSupport

A two-tool pipeline for importing Google Earth survey data into AutoCAD. KMZ Processor (VBA) handles data extraction and transformation; ImportSupport (AutoLISP) consumes its output to place objects directly in a drawing.

  • KMZ Processor converts Google Earth KMZ files into CSVs with objects separated by category, using XPath to parse and traverse the KMZ structure
  • Additional operations include splitting a KMZ into user-drawn areas, and aggregating and summarizing valid data within it
  • ImportSupport reads the CSV output and places coordinate-mapped objects in AutoCAD, completing the pipeline

Network Design Tools — AutoLISP Routines

A collection of 15 AutoLISP routines built to address inefficiencies in the OSP design workflow.

  • FixPDFImport: Fixes missing geometry from PDF imports, avoiding manual fixes
  • DrawFiber: Speeds up schematic creation by automating multi-step drawing tasks
  • Additional routines include: centering a polyline between two others, callout generation from object data, and automatic distance measurements

DesignShark — Fiber Design Automation Suite

A growing suite of automation tools built in Excel / VBA, adopted by multiple teams of contractors at NWA and Charter.

  • Automates repetitive device naming, data formatting, and deliverable generation, saving 6+ hours per design (25%+ efficiency gain)
  • Handles ingestion and processing of exports from Magellan, Iris, and Google Earth
  • Evolved iteratively over time from a single script into a full multi-tool suite based on team feedback

Fiber Design QC Tool

  • Built with a teammate to validate design deliverables for completeness and consistency/li>
  • Compares user data to reports from Magellan and Google Earth, runs automated checks, and calls out errors in a dashboard
  • Drastically reduces QC rejections for design teams using the tool
  • Allows QC teams to focus on deeper qualitative investigations, since quantitative checks are handled by the tool

Omnigon
